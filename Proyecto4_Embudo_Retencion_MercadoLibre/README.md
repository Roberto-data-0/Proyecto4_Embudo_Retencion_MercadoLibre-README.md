# Proyecto 4 – Análisis de embudo y retención para Mercado Libre

## 🎯 Objetivo
Analizar el comportamiento de los usuarios en la plataforma de Mercado Libre, evaluando el embudo de conversión y las métricas de retención.

## 📂 Dataset
- Fuente: [indica si fue proporcionado por el bootcamp, Kaggle, etc.]
- Tamaño: [ejemplo: 200,000 registros]
- Variables principales: visitas, clics, carritos, compras, usuarios activos, fechas.

## 🔧 Herramientas utilizadas
- SQL para consultas de cohortes y embudo.
- Python (Pandas, Matplotlib/Seaborn) para análisis y visualización.
- Power BI para dashboards interactivos.

## 📊 Pasos principales
1. Definición de etapas del embudo: visita → clic → carrito → compra.
2. Cálculo de tasas de conversión por etapa.
3. Análisis de cohortes de usuarios para medir retención.
4. Visualización de métricas en dashboard.

## 📈 Resultados
- Tasa de conversión general del embudo.
- Identificación de la etapa con mayor abandono.
- Métricas de retención por cohortes mensuales.
- Insights sobre comportamiento de usuarios recurrentes.

## 💡 Aprendizajes
- Cómo aplicar análisis de embudo en e-commerce.
- Uso de cohortes para medir retención de clientes.
- Valor de combinar SQL, Python y BI para un análisis integral.
